# 完整工程骨架与 MVP 闭环实现 [Project Skeleton with MVP Runnable Slice]

本项目采用完整工程骨架 [Project Skeleton] 加 MVP 闭环实现 [MVP Runnable Slice] 的交付方式：目录、前端路由 [Frontend Routes] 与占位页面 [Placeholder Pages] 可以为 Employees、Shippers、Categories、Suppliers、System 等后续模块预留入口，但后端只实现最小可运行后台 [Runnable MVP Admin] 所需 API。

这样做保留了二次开发 [Further Development] 的结构可见性，同时避免为未确认业务规则 [Business Rules] 的模块创建空服务 [Empty Services] 或伪实现 [Stub Implementations]。README 必须明确区分已完成 [Implemented] 与预留 [Reserved] 模块。
